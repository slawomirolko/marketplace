# Olko Kotlin Architecture Overview

Checks Kotlin/Android architecture for changed `.kt`/`.kts` files with marketplace defaults: source-set layout, module boundaries, dependency direction, MVVM/ViewModel-StateFlow separation, networking singleton ownership, Gradle dependency configuration, security boundaries, and external-service client configuration. Use before commit/test gates, during plan review, or when delegated by olko-commit-style/olko-test/olko-plan-editor.

Load `SKILL.md` for the canonical workflow and rule list.
