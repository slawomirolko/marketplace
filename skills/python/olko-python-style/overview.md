# Olko Python Style Overview

Checks Python style for changed `.py` files with marketplace defaults: functions/type hints, generated-code ownership, file creation (`pathlib`, explicit encoding), single responsibility, and zero-tolerance silent-failure rules. Runs documented format/lint/type commands (ruff/mypy/pyright) when configured. Use before commit/test gates or when delegated by olko-commit-style.

Load `SKILL.md` for the canonical workflow and rule list.
